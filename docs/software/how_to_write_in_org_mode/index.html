    • David O'Toole Org tutorial

    Headings¶

    ***** Heading Level 1
****** Heading Level 2
******* Heading Level 3
******** Heading Level 4
etc.
    • Headings can be inserted at the same level by Alt + Enter from end of line.
    • Headings can be rearranged up or down by Alt + ↑/↓.
    • Headings can be changed to another level by Alt + ←/→.

    Lists¶

    - Item 1
- Item 2
    • Lists are very similar to headings. The shortcuts for rearranging heading above can also be used for rearranging lists.
    • List style can be changed by Shift + ←/→.

    Commonly-used symbols¶

    Insert the following into your .emacs file:

    ;; Set up keyboard shortcuts for math symbols
;; http://ergoemacs.org/emacs/emacs_n_unicode.html
(define-key key-translation-map (kbd "<f9> a") (kbd "α"))
(define-key key-translation-map (kbd "<f9> b") (kbd "β"))
(define-key key-translation-map (kbd "<f9> d") (kbd "δ"))
(define-key key-translation-map (kbd "<f9> D") (kbd "Δ"))
(define-key key-translation-map (kbd "<f9> ul") (kbd "µl"))
(define-key key-translation-map (kbd "<f9> ug") (kbd "µg"))
(define-key key-translation-map (kbd "<f9> ->") (kbd "→"))
(define-key key-translation-map (kbd "<f9> <-") (kbd "←"))

    Then you can hit F9 and the key above to insert the corresponding symbol.

    Tables¶

    | Col 1  | Col 2  | Col 3  |
|--------+--------+--------|
| Data 1 | Data 2 | Data 3 |
| Data 4 | Data 5 | Data 6 |

    Note that the tables are smart:

    • Tab will introduce a new column.
    • Enter will introduce new now.
    • |- followed by Tab will introduce the full line between header and first row.

    Automatic timestamps¶

    • Ctrl-C → Shift-1 will introduce date stamp.
    • Ctrl-U → Ctrl-C → Shift-1 will introduce date with time stamp.

    TODO items¶

    You can make any heading into a TODO item by Shift + →. Pressing this again, will cycle the TODO state to WAITING, DONE etc.

    Deadlines and Scheduling¶

    • Ctrl + c → Ctrl + s from a headline will insert a schedule date for that heading.
    • Ctrl + c → Ctrl + d from a headline will insert a deadline date for that heading.

    Figures¶

    You can insert links to images with [[file:PATH_TO_IMAGE] and then display them within the Org mode buffer with the command M-x org-display-inline-images.

    This will display the image in its full size (usually way too large), so you can either resize the image in Inkscape, Gimp etc. before linking, or add some code to .emacs to set a default display width. The second option is OS-specifc.

    Source code¶

    #+BEGIN_SRC

    Source code here

    #+END_SRC

    Keyboard shortcut is <s → Tab.

    You can also get language-specific syntax highlighting when you export Org mode files to HTML or Latex. Look at Rasi's Emacs configuration file: https://github.com/rasilab/rasi_xps15_config/blob/master/emacs#L225-L244 for how to do this.

    Latex¶

    Most latex syntax can be used without any special notation in Org mode. This is particularly useful for fine control over figure placements, journal-specific environments etc.

    To insert latex headers, do

    #+LATEX_HEADER: \usepackage{amsmath}

    Insert table of contents into your Org mode file¶

    It is really useful to have a Table of Contents for Org files that you frequently view on Github.

    You can install TOC-Org package so that every time you save your Org file, a Table of Contents is automatically generated and updated.

    To install this package, do M-x package-install and enter toc-org.

    Then insert a :TOC: tag to the first headline in the Org file by C-c C-q. Look at the Raw code for this file as an example.

    The above TOC was generated automatically using the TOC-Org package.

    Work with rectangular regions¶

    Rectangular cut, copy, paste is useful for working with tabular data.

    You can select rectangular regions by setting a mark at one diagonal using C-SPC, and then going to the other diagonal point of the rectangle.

    C-x r k will cut the rectangular region.

    C-x r y will paste the rectangular region.

    For more information, see https://www.emacswiki.org/emacs/RectangleCommands.
